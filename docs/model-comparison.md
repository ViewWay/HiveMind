## HiveMind vs 开源模型

### Qwen3.5 系列

| 模型 | 参数 | MMLU | HumanEval | GSM8K | 来源 |
|------|------|------|-----------|-------|------|
| Qwen3-4B | 4B | 76.6% | - | - | [Qwen](https://github.com/Qwen) |
| Qwen3.5-35B-A3B | 35B/3B | 87.8% | 76.4% | 92.7% | [Qwen](https://github.com/Qwen) |
| Qwen3.5-397B | 397B/17B | 88.4% | - | - | [Qwen](https://github.com/Qwen) |
| **HiveMind (目标)** | **207B/25M** | **70%** | **50%** | **65%** | 本项目 |

**Qwen3.5 基线分析**:
- Qwen3.5 使用了 MLA (Multi-head Latent Attention) 减少 KV Cache
- 采用 A3B (Activate 3 Billion) 稀疏激活策略
- HiveMind 参考 Qwen 的 MoE 设计，但针对小规模优化

### GLM-5 (Zhipu AI)

| 指标 | GLM-5 | HiveMind | 对比 |
|------|-------|----------|------|
| 总参数 | 744B | 207M | 0.03% |
| 激活参数 | 40-44B | 25-42M | 0.1% |
| MoE 架构 | 256专家/8激活 | 8-16专家 | 简化版 |
| AIME 2026 | 92.7% | - | 目标中 |
| MMLU | - | 70% | 精简版 |
| 上下文 | 200K | 2-4K | - |

**GLM-5 设计参考**:
- 训练数据规模: 28.5T tokens
- DeepSeek Sparse Attention (DSA) 集成
- 异步强化学习基础设施 (Slime)
- HiveMind 学习其稀疏激活策略

### DeepSeek-V3

| 特性 | DeepSeek-V3 | HiveMind | 借鉴 |
|------|-------------|----------|------|
| MLA | ✅ | - | 考虑引入 |
| MoE | 256专家 | 8-16专家 | 规模适配 |
| 训练数据 | 数T tokens | 百万级 | 分阶段 |

## 技术路线图

### 架构演进

```
Phase 1: 当前 (207M)
├── 8 专家 × 25M 参数
├── 固定 Top-K 路由
└── 简单损失函数

Phase 2: 优化 (500M)
├── 16 专家 × 30M 参数
├── 自适应混合路由
├── MLA (参考 Qwen)
└── 负载均衡损失

Phase 3: 高级 (1B)
├── 32 专家 × 32M 参数
├── DeepSeek Sparse Attention
├── 异步训练基础设施
└── 思维链模式
```

### 训练策略

| 阶段 | 数据量 | 对比 GLM-5 |

| 阶段3 | 混合任务 | 路由训练 |

## 关键优势

### 与 Qwen3.5 相比

1. **本地可运行**: M4 Pro 即可运行，无需云服务
2. **可定制**: 完全开源，可自由修改和扩展
3. **轻量级**: 仅 207M 参数 vs Qwen 的 35B+

### 与 GLM-5 相比

1. **资源友好**: 训练和推理成本极低
2. **快速迭代**: 小模型训练周期短，便于实验
3. **边缘部署**: 可部署到本地设备

### 与 DeepSeek-V3 相比

1. **简化架构**: 保留核心 MoE 思想，简化实现
2. **渐进式训练**: 分阶段降低训练门槛
3. **教育价值**: 代码清晰，适合学习和研究
