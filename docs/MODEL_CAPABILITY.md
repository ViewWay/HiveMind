## ⚠️ 局限性

### 1. 数据量限制
```
当前: 152 条
推荐: 10,000+ 条 (专业微调)
      100,000+ 条 (通用模型)
```

### 2. 领域限制
```
✓ AI/机器学习/深度学习  → 训练过，效果较好
○ 编程/数学/物理        → 基础模型能力，未微调
✗ 文学/历史/日常对话     → 表现较差
```

### 3. 能力天花板

LoRA 微调 **不能** 超越基础模型的能力，只能：
- 调整知识分布
- 改变输出风格
- 强化特定领域
- 修正错误知识

## 🚀 快速提升方案

### 方案 1: 增加数据（最简单）
```bash
# 收集 1000+ 条相关文本
# 重新训练
PYTORCH_MPS_HIGH_WATERMARK_RATIO=0.0 uv run python train.py
```

### 方案 2: 优化配置
```python
# 修改 train.py
"lora_r": 16,              # 增加表达能力
"target_modules": ["q_proj", "k_proj", "v_proj", "o_proj", 
                   "gate_proj", "up_proj", "down_proj"],  # 全部层
"num_train_epochs": 5,     # 训练更久
```

### 方案 3: 使用更强的配置
```bash
# 对比实验找出最佳配置

```

## 📌 关键要点

1. **LoRA 微调 ≠ 从零训练**
   - 只调整 0.0374% 的参数
   - 依赖基础模型的能力
   
2. **数据质量 > 数据数量**
   - 100 条高质量 > 1000 条低质量
   - 多样性很重要
   
3. **领域专注是优势**
   - 通用模型需要海量数据
   - 专用模型可以用少量数据做好

4. **持续迭代**
   - 先用少量数据验证流程
   - 逐步增加数据优化效果
